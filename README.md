## 🧭 프로젝트 개요

우리 4조(밥사조)는 **(프로젝트 이름)** 을(를) 통해, **API 기반 자동화된 TRPG 세션 진행 서비스**를 제안한다.  
본 프로젝트는 단순한 대화 생성/캐릭터 채팅이 아니라, **룰 기반 판정 시스템**과 **서사 전개**가 유기적으로 결합되는 TRPG의 핵심 구조를 디지털 환경에서 안정적으로 재현하는 것을 목표로 한다.

## 📚 TRPG란 무엇인가

**TRPG(Tabletop Role-Playing Game)** 는 참가자들이 각자 하나의 캐릭터를 맡아, 정해진 **규칙(rule)** 과 **시나리오(scenario)** 를 기반으로 공동의 이야기를 만들어가는 놀이 형태이다.  
일반적인 비디오 게임과 달리, TRPG에서는 플레이어의 선택이 고정된 분기점에 한정되지 않으며, **플레이어의 발화와 판단에 따라 서사가 유동적으로 변화**한다는 특징을 가진다.

TRPG 진행에는 보통 **수호자(Game Master, GM)** 가 필요하다. GM은 시나리오를 제시하고, 플레이어의 행동에 따라 상황을 묘사하며, 규칙에 따른 **판정**과 **결과 해석**을 담당한다.  
플레이어는 자신의 캐릭터 **스탯**과 성격을 바탕으로 행동을 선언하고, **주사위 굴림**을 통해 성공 여부를 판정받는다. 이 과정에서 TRPG는 게임적 요소(룰, 스탯, 확률)와 서사적 요소(대화, 묘사, 즉흥극)가 동시에 작동하는 복합적인 구조를 가진다.

또한 TRPG의 핵심은 ‘승패’보다는 **세션의 완결**과 **서사의 경험**에 있다. 하나의 세션은 일정한 서사적 목표를 중심으로 진행되며, 플레이어는 캐릭터에 이입한 상태에서 문제를 탐색하고, 단서를 수집하며, 사건을 해결하거나 파국을 맞이한다.  
특히 **탐사 중심 TRPG**의 경우, 전투보다 정보 획득과 판단 과정이 중요한 비중을 차지한다.

이러한 특성으로 인해 TRPG는 단순한 대화형 놀이와 구분된다. TRPG에서의 발화는 자유로운 채팅이 아니라, **규칙과 상황 맥락을 전제로 한 ‘행동 선언’** 이며, 모든 선택은 이후의 서사와 게임 상태에 영향을 미친다.  
즉 TRPG는 연속적인 대화의 축적이 아니라, **규칙에 의해 제어되는 상태 변화의 연쇄**라고 볼 수 있다.

> ✅ 요약: TRPG는 **룰 기반 판정 시스템 위에서 서사와 즉흥극이 결합된 게임 형식**이며, GM의 판단과 관리 능력이 세션의 질을 크게 좌우한다.

## 🧩 문제 정의

TRPG는 보드게임과 마찬가지로 여러 사람과 같은 시공간을 공유하며 연속적인 세션을 즐기는 행위이다. 다시 말해 TRPG를 즐기기 위해서는 **물리적 자원**과 **인적 자원**이 필요하다.

그러나 세션을 함께할 사람을 찾는 것은 쉽지 않다. TRPG는 단순한 게임 플레이가 아니라 즉흥극의 형식을 차용하기 때문에 **캐릭터 및 서사 취향**, **유저 간 성향**의 영향을 크게 받는다.  
설령 함께 플레이할 사람을 찾았더라도, **일정 조정** 또한 어렵다. 한 세션은 암묵적으로 1시간 이상을 요구하며, 그 시간 동안 플레이어는 캐릭터로 이입해 즉흥극을 수행한다.

특히 수호자는 세션을 관리·감독하면서 플레이어가 시나리오대로 움직이고 있는지, 현재 상황이 어떤지 등 종합 정보를 처리해야 한다. 또한 **KPC**가 필요한 시나리오에서는 수호자가 위 역할을 수행하면서 동시에 캐릭터로 이입해 역극을 발화·작성해야 하므로 피로도가 매우 높다.  
텍스트 채팅 위주 세션에서는 소설/희곡에 가까운 장문의 글을 주고받아 **읽기/쓰기 부하** 또한 커진다.

## 🎯 프로젝트 목표

(프로젝트 이름)의 목표는 다음과 같다.

### 1) CoC 7판 단일 룰북 채택
Call of Cthulhu 7판(CoC 7th)을 단일 룰북으로 채택하여, 특정 룰에 기반한 **일관된 TRPG 세션**을 안정적으로 제공한다.  
범용 규칙 기반 서비스와 달리, 룰북에 명시된 **스탯 구조와 판정 방식**을 그대로 반영하여 탐사·공포 중심 TRPG의 게임성을 유지한다.

### 2) 룰북 기반 텍스트 생성 구조 설계
판정 결과와 상태 변화가 서사 전개와 유기적으로 결합되도록 텍스트 생성 구조를 설계한다.  
이는 단순 대화 생성이 아니라, **규칙 기반 게임 플레이를 전제로 한 서사 생성**이라는 점에서 기존 캐릭터 채팅과 구분된다.

### 3) TRPG 전용 프론트 설계
단순 채팅 UI를 탈피하여, 비선형적 시나리오 진행과 **스탯 관리·확인**을 용이하게 하는 TRPG 전용 프론트를 설계한다.  
이를 통해 사용자는 현재 상황과 선택이 세션 전체에서 갖는 의미를 직관적으로 파악할 수 있다.

### 4) 팬메이드 시나리오 업로드 기반 진행
사용자가 직접 작성하거나 선택한 시나리오 텍스트를 업로드하면 이를 기반으로 세션이 진행되도록 하여, TRPG 고유 문화인 **팬메이드 시나리오 활용**을 디지털 환경에서도 유지한다.  
이는 플랫폼 종속적 시나리오 소비 구조에서 벗어난다는 점에서 의의가 있다.

### 5) 세션 진행 기복 감소 및 시간 단축
API 기반 **수호자(GM) 및 KPC 역할** 수행을 통해 세션 진행의 기복을 줄이고, 텍스트 생성 속도를 개선하여 전체 세션 소요 시간을 단축한다.  
사용자는 물리적 공간이나 동시 접속 인원에 구애받지 않고, 원할 때 언제든 세션을 시작하고 완결할 수 있다.  
이는 chit-chat 중심 서비스와 달리 **세션 완결을 지향하는 task-oriented 서비스**라는 점에서 차별성을 가진다.

## 🏗️ 개발 전략 및 범위(제약 조건)

본 프로젝트는 기능 확장보다 **세션 단위 TRPG가 온전히 성립하는 최소 조건을 검증**하는 데 우선순위를 두었다.  
즉, 범용 TRPG 플랫폼을 즉시 완성하기보다 **하나의 TRPG 세션 사이클이 처음부터 끝까지 안정적으로 작동함**을 1차 성과로 설정하였다.

<details>
<summary><b>✅ 제약 조건(펼쳐보기)</b></summary>

### 1) 1회 완결형 단일 세션 구조
세션 시작–진행–판정–종료 흐름이 끊기지 않고 유지되는지 검증한다. 장기 캠페인/다회차 누적 세션 기능은 범위에서 제외했다.

### 2) 최소 모듈 + 단일 시나리오
룰북은 CoC 7판으로 한정하고, 검증 목적에 적합한 짧고 단순한 단일 시나리오를 사용한다.  
룰 해석, 판정 처리, 서사 생성의 연동이 올바르게 작동하는지 집중 검증한다.

### 3) 등장인물 1:1(플레이어 1명 + KPC 1명)
1:1 TRPG 환경에서 GM과 KPC 역할이 동시에 요구되는 상황을 가정한다.  
LLM은 GM과 KPC의 역할을 동시에 수행하며, 단일 API 호출 구조 안에서 복수 역할을 안정적으로 유지하는지 검증한다.  
장르는 비교적 상호작용이 단순한 로맨스(또는 로맨스 요소 포함 서사)로 제한한다.

### 4) 게임 요소 최소화
주사위 판정과 스탯 기반 성공·실패 판단은 포함하되, 복잡한 전투 규칙이나 반복 판정 중심 구조는 배제한다.  
초점은 전투 시뮬레이션이 아니라, **룰 기반 판정이 서사 생성과 어떻게 결합되는지** 검증하는 데 있다.

### 5) 로컬 기반 실행(서버 운영 제외)
별도의 서버 인프라 없이 로컬 환경에서 API 호출로 세션을 진행한다.  
인터넷 연결은 필요하지만 서버 관리/동시 접속 처리는 범위에서 제외한다.

### 6) 3시간 이내 완료 가능한 분량 제한
세션 완결이라는 목표를 명확히 드러내기 위해 3시간 이내에 완료 가능한 분량으로 제한한다.  
탐사·판단·결말이 포함된 최소 TRPG 경험 제공을 목표로 한다.

</details>
